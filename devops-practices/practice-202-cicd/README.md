# LinkShort — Сервис сокращения ссылок

## Описание проекта

LinkShort — учебное веб-приложение для сокращения URL-адресов.

Пользователь сможет:
- вводить длинный URL и получать короткую ссылку;
- переходить по короткой ссылке (редирект на оригинальный адрес);
- просматривать статистику переходов по своим ссылкам;
- удалять созданные ссылки.

Проект создаётся в рамках курса «Программная инженерия (Часть 2): Введение в DevOps».

## Результаты аудита потока (AS-IS)

Анализ проводился на основе работы над учебными проектами в рамках курса.

**Lead Time (от идеи до работающего кода):** 5–7 дней.

**Главные боли процесса:**

1. **Ожидание code review** — PR может висеть без проверки 1–2 дня; нет SLA и автоматических проверок, которые ускоряли бы принятие решения.
2. **Ручной деплой** — публикация обновлений выполняется вручную: подключение к серверу, остановка контейнеров, pull, перезапуск. Высокий риск ошибки и потери времени (~30–60 мин на каждый деплой).

Подробный анализ — в `documentation/value_stream_map.md`.

## Стратегия ветвления

Используется **Trunk-Based Development**.

Ветки:
- `main` — защищённая основная ветка, только через PR
- `feature/*` — новая функциональность
- `fix/*` — исправления

## Git Governance

Для ветки `main` настроены Branch Protection Rules:
- прямой push запрещён;
- изменения принимаются только через Pull Request;
- обязателен минимум 1 approve;
- CI должен пройти успешно перед merge.
