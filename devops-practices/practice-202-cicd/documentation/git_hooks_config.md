# Конфигурация Git Hooks (pre-commit)

## Описание

`pre-commit` — фреймворк для управления локальными Git-хуками.  
Хуки запускаются **до** создания коммита и блокируют его при нарушении правил.  
Это позволяет поймать проблемы ещё до попадания кода в репозиторий.

---

## Установленные хуки

| Хук                    | Инструмент          | Что проверяет                                         | Польза                                                    |
|------------------------|---------------------|-------------------------------------------------------|-----------------------------------------------------------|
| `ruff`                 | ruff                | Синтаксис и стиль Python-кода (PEP8)                 | Единый стиль кода в команде, меньше замечаний на ревью    |
| `black`                | black               | Форматирование кода                                   | Автоматически форматирует файл, не нужно делать вручную   |
| `detect-secrets`       | detect-secrets      | Поиск секретов (пароли, токены, ключи API)           | Предотвращает случайный коммит credentials в репозиторий  |
| `check-yaml`           | pre-commit-hooks    | Валидность YAML-файлов                               | Ловит синтаксические ошибки в конфигах до пуша            |
| `check-json`           | pre-commit-hooks    | Валидность JSON-файлов                               | Предотвращает сломанные конфиги                           |
| `end-of-file-fixer`    | pre-commit-hooks    | Наличие переноса строки в конце файла                | Совместимость между редакторами и ОС                      |
| `trailing-whitespace`  | pre-commit-hooks    | Лишние пробелы в конце строк                         | Чистые диффы в PR, меньше шума при ревью                  |

## Инструкция для новых контрибьюторов

### 1. Установить pre-commit

```bash
pip install pre-commit
```

### 2. Активировать хуки в репозитории

```bash
# Выполнить один раз после клонирования репозитория
pre-commit install
```

### 3. Проверить работу на всех файлах

```bash
pre-commit run --all-files
```

### 4. Работа в обычном режиме

После установки хуки запускаются автоматически при каждом `git commit`.  
Если проверка не прошла — коммит блокируется, файлы исправляются автоматически (где возможно).  
Нужно добавить исправленные файлы и повторить коммит:

```bash
git add .
git commit -m "your message"
```
