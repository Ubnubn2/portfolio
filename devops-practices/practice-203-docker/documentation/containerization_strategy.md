# Стратегия контейнеризации

## Выбор базового образа

Для сборки используется:

python:3.12-alpine

Причины выбора:

* небольшой размер образа;
* минимальное количество лишних пакетов;
* быстрое скачивание;
* снижение поверхности атак.

Для PostgreSQL используется:

postgres:16-alpine

Для Redis используется:

redis:7-alpine

---

## Переменные окружения

| Переменная  | Назначение       |
| ----------- | ---------------- |
| DB_HOST     | Хост базы данных |
| DB_PORT     | Порт PostgreSQL  |
| DB_NAME     | Имя БД           |
| DB_USER     | Пользователь БД  |
| DB_PASSWORD | Пароль БД        |
| REDIS_HOST  | Хост Redis       |
| REDIS_PORT  | Порт Redis       |

---

## Multi-stage Build

Используется двухэтапная сборка:

### Build Stage

Установка зависимостей и подготовка приложения.

### Runtime Stage

Содержит только необходимые файлы для запуска.

---

## Результаты оптимизации

Размер образа до оптимизации:

≈ 650 МБ

Размер образа после Multi-stage Build:

≈ 120 МБ

Снижение размера:

≈ 80%

Использованные методы:

* Alpine Linux
* Multi-stage Build
* Удаление кэша pip
* Минимальное количество слоёв
